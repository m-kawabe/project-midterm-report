まず、従来行われている取り組みを調べることが重要であると考えた。そして一般的な開発の流れを参考にし、以下のように手順を設定した。

\begin{enumerate}
  \item 従来の取り組みの調査
    \par 課題：「ヘルスツーリズム」の取り組みはどのように行われているかを調査する。他の都市ではどのような町歩きマップを作成しているかを調査する。他のウォーキングアプリはどのような機能によってヘルスケアを支援しているかを調査する。
  
  \item 要件定義
  \begin{enumerate}
     \item ターゲットユーザの設定
     \par 課題：観光客と函館市民どちらをターゲットとするのかを検討する。
  
     \item 開発するアプリのコンセプトの設定
     \par 課題：コンセプトの類似例の調査する。
  
     \item 機能の考案
     \par 課題：考案した機能を図に記載する。
  
     \item 考案した機能の検証
     \par 課題：考案した機能のシナリオの作成する。
  \end{enumerate}
  
  \item データ利用の許可を得る
  \par 課題：「健康ウォーキングマップ」を提供している函館市保険福祉部健康増進課にデータ利用許可を得る。
  
  \item 設計
  \par 課題：各画面の詳しい仕様や画面の遷移、機能の仕様の設計を行う。
  
  \item 実装
  \par 課題：設計で決定したことを実現するためにiOSにて実装を行う。プログラミング言語はObjective-Cを使用する。
  
  \item テスト
  \par 課題：作成したアプリが仕様通りに動くか、バグはないかといったテストを行う。
  
  \item 運用保守
  \par 課題：一般向けにリリースを行う。リリース後もアプリに問題はないかといったことをチェックする。
\end{enumerate}
